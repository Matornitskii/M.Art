# M.Art — лаборатория экспериментов

M.Art — это статический сайт, где каждая страница может быть отдельным экспериментом: визуализацией, тестом, интерактивом, мини-игрой или прототипом идеи.

## Что уже реализовано

- Главная страница-каталог с автогенерацией карточек экспериментов из `data/experiments.json`.
- Базовая страница «О проекте».
- Страница 404 для несуществующих URL.
- Два примера экспериментов:
  - **Сколько дней осталось жить** — расчёт и визуализация условного жизненного времени.
  - **Лаборатория мини-тестов** — мини-опрос, тест реакции, логический тест с сохранением данных в `localStorage`.

## Структура проекта

```text
.
├── index.html                  # Главная страница-каталог
├── about/index.html            # Страница «О проекте»
├── 404.html                    # Страница ошибки
├── data/experiments.json       # Манифест экспериментов (источник каталога)
├── assets/
│   ├── css/base.css            # Общие стили каталога и служебных страниц
│   └── js/catalog.js           # Логика автогенерации каталога
└── experiments/
    ├── life-days/
    │   ├── index.html
    │   ├── life-days.css
    │   └── life-days.js
    └── tests-lab/
        ├── index.html
        ├── tests-lab.css
        └── tests-lab.js
```

## Как запустить локально

> Важно: открывать `index.html` двойным кликом не рекомендуется, потому что загрузка JSON через `fetch` может блокироваться в режиме `file://`.

1. Откройте терминал в корне проекта.
2. Запустите статический сервер:

```bash
python -m http.server 8000
```

3. Откройте в браузере: `http://localhost:8000`.

## Как добавить новый эксперимент

1. Создайте новую папку, например `experiments/my-new-idea/`.
2. Добавьте туда минимум `index.html` и при необходимости отдельные `css/js` файлы.
3. Добавьте запись в `data/experiments.json`:

```json
{
  "id": "my-new-idea",
  "title": "Название",
  "description": "Краткое описание",
  "status": "черновик",
  "url": "./experiments/my-new-idea/",
  "visible": true
}
```

4. Обновите страницу каталога — карточка появится автоматически.

### Как скрыть эксперимент без удаления

- Поставьте `"visible": false` в `data/experiments.json`.
- Эксперимент останется в репозитории, но пропадёт с главной страницы.

## Публикация через GitHub Pages

1. Создайте репозиторий на GitHub и загрузите проект.
2. Перейдите в **Settings → Pages**.
3. В блоке **Build and deployment** выберите:
   - **Source**: `Deploy from a branch`
   - **Branch**: `main` (или ваша рабочая ветка), папка `/root`
4. Сохраните настройки.
5. Через 1–3 минуты сайт будет доступен по ссылке вида:
   - `https://<username>.github.io/<repo-name>/`

## Почему эта архитектура удобна для роста

- Каталог не нужно редактировать вручную при каждом новом эксперименте.
- Каждый эксперимент изолирован в своей папке и может иметь собственный стиль/логику.
- Черновики можно держать в коде, но отключать от каталога флагом `visible`.
- Проект полностью статический и не зависит от сервера или БД.
